# Websites & references

Tags: `[level] [domain] [free]`

- **OpenRouter model catalog** — openrouter.ai/models — `[all] [models] [free]` — Why: live prices and context windows for 300+ models; the always-current companion to our frozen [matrix](../decision-support/model-matrix.md).
- **Hugging Face** — huggingface.co — `[builder] [models/datasets] [free]` — Why: the home of open-weights models; model cards teach you to read capabilities and licenses.
- **Prompt Engineering Guide** — promptingguide.ai — `[novice→builder] [prompting] [free]` — Why: structured, cited guide from basics to advanced patterns; better than scattered tips threads.
- **RAGAS documentation** — docs.ragas.io — `[builder] [evaluation] [free]` — Why: the evaluation framework used in Session 2; the concepts page alone teaches what "good RAG" measurably means.
- **LangChain / LangGraph docs** — python.langchain.com — `[builder] [orchestration] [free]` — Why: reference for Sessions 1–2 stacks; start with the conceptual guides, not the API reference.
- **Google AI Studio** — aistudio.google.com — `[all] [hands-on] [free]` — Why: the most generous no-card free tier (see matrix) — your zero-cost lab bench.

**Check yourself:** open the OpenRouter catalog, find one model NOT in our matrix, and decide which row it would replace. (This is the matrix-maintenance skill you'll need after this repo goes stale.)
